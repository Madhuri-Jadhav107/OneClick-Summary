### Authentication Method

### Why Firebase?
- Robust, secure identity management
- Easy integration with Firestore (NoSQL database)

## ğŸ§‘ User Identity Management

### `user_id` (uid)
- Generated by **Firebase** at signup
- Unique identifier for the user (`user.uid`)
- Used to secure access to meeting data

### User Profile
- Additional user details are stored in **Cloud Firestore** under the `users` collection.
- Document ID matches the Auth `uid`.

Example User Document (`users/{uid}`):
```json
{
  "email": "user@company.com",
  "full_name": "John Doe",
  "company_id": "company_101",
  "role": "user",
  "created_at": "2023-10-27T10:00:00Z"
}
```
